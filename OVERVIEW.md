## 1. 設計思想

### 根本原則：会社と同じ組織をAIで再現する

HAMIGAKIという会社が持つべき部門を、そのままAIエージェントで構成する。
「AIツールを使う」のではなく、「AIで会社組織を回す」。

代表（佐野）がやることは3つだけ：
1. **方向性を決める**
2. **質問に答える**
3. **成果物を承認する**

それ以外の実行は全部AIエージェントがやる。

### リオ氏（空気デザイン）の運用設計から借りた原則

1. **専門化** — 1体1役割。全部やるAIは中の上で止まる
2. **レビュー文化** — 複数の目を通す。1体のAIの出力をそのまま使わない
3. **書き物で引き継ぎ** — AI同士は会話しない。Notionに書いて非同期で読む
4. **権限の明示** — ここまではAIが決めていい、ここからは人間が判断する

### リオ氏との決定的な違い

| | リオ氏（空気デザイン） | HAMIGAKI |
|---|---|---|
| CEOの関与 | 1日12時間AIに張り付く | 朝のSlack確認＋判断だけ |
| 指揮系統 | CEO → 各チームリーダー直接 | CEO → 秘書AI＋COO AI → 各部門Dir |
| 24時間稼働 | クラウドAI中心 | ローカルLLMで24時間稼働（コスト0） |
| CEOの窓口 | 全チームと直接やり取り | 秘書AIとCOO AIの2人だけ |

## 4. タスクの流れ

```
代表 or 秘書AIが指示
  ↓
INBOX（Notion）
  ↓ タスクディスパッチャー（ローカルLLM）
適切な部門のDirに振り分け
  ↓ 各部門Dir（Sonnet）がチーム内に割り当て
AI作業中
  ↓ 実行AI（Sonnet/Haiku）が成果物作成
  ↓ 品質チェック（ブランドボイス・アンチAIスロップ）
👀代表確認 ← 成果物リンク＋確認ポイント付き
  ↓ Slack朝ブリーフィングで通知
  ↓ 代表がレビュー
承認 → 完了 / 差し戻し → AI作業中に戻る
```

### タスクDBのステータス

| ステータス | 色 | 意味 |
|---|---|---|
| INBOX | default | 未整理。ディスパッチャーが振り分ける |
| Backlog | gray | いつかやる |
| 1週間以内 | green | 今週中に着手 |
| 今日やる | yellow | 今日やるべき |
| 🔥NOW | red | 今すぐやっている |
| AI作業中 | blue | AIエージェントが実行中 |
| 👀代表確認 | purple | AI完了。代表のレビュー待ち |

## 6. COO AI 育成フェーズ

### フェーズ1（今〜1ヶ月目）: 記録する

代表がCOOの役割をやりながら、全部Notionに記録する。

- 意思決定のたびに「何を、なぜ、どう決めたか」を意思決定ログに残す
- 「これは自分じゃなきゃダメ」「これは任せていい」の線引きを残す

**この期間に動かすもの:**
- 秘書AI（既存・継続）
- タスクディスパッチャー（ローカルLLM）
- 各部門のDir AI（Sonnet）← 代表が直接指示

**成果物:** 意思決定ログ30件以上 + 判断基準ページ

### フェーズ2（2ヶ月目）: 隣に置く

COO AIを「提案係」として起動。

- 意思決定ログと価値観ページを読ませた上で提案させる
- 「違う、こうだ」と直すたびに修正理由を追記
- 正答率70%以上で次のフェーズへ

### フェーズ3（3ヶ月目〜）: 権限を渡す

段階的に権限を広げる：

| 段階 | COOが自分で決めていい範囲 |
|---|---|
| 3-1 | P2以下のタスク振分 |
| 3-2 | コンテンツの修正指示 |
| 3-3 | チーム間の調整・ブロック解消 |
| 3-4 | P1タスクの優先順位づけ |
| 3-5 | P0以外のすべて |

**P0（常に代表判断）:**
- 事業方向性の変更、大型案件の受諾/辞退、予算決裁
- 対外コミットメント、エージェント指示書の変更

**フェーズ3後半:** ローカルLLMへの移行を検討。意思決定ログが教科書になる。

## 11. リオ氏の「1000時間の知見」を反映した注意事項

### 暗黙の了解は通用しない
やっていいこと・やってはいけないことを全部書き下ろす。
「消すなと書いてなければ消す可能性がある」（リオ氏の実体験）
→ 今日、Notionの営業管理ページをアーカイブしてしまった失敗がまさにこれ。

### 静かに壊れる
AIは「壊れたファイルに書き込み完了しました」と返して次に行く。
→ 👀代表確認のステータスと引き継ぎノートで「何をやって何が残っているか」を必ず可視化する。

### 指示書の精度 = 組織の品質
指示書を省略すると必ず事故が起きる。
→ 各部門Dir指示書は、🦷ページのような業務マニュアルとセットで機能する。

### 作り込みすぎると進まない
代表は細かく決めがちで先に進まなくなる傾向がある（本人も自覚あり）。
→ AI側から「重箱の隅を突いている」と感じたら意見する。80点で出して走りながら直す。
