## 🦷 SCHRAIBER.AI とは（コアコンセプト）

**一言で：** 歯科診療の「書く時間」を、ゼロにする AI音声カルテシステム。

- **形態：** 歯科診療に特化したクラウドAIサービス
- **入力：** 診療中の医師・スタッフの会話（音声）
- **出力：** SOAP形式の診療記録（テキスト）
- **使い方：** ①録音開始 → ②AI自動整形 → ③確認して既存カルテに貼り付け
- **思想：** 基幹カルテシステムを置き換えるのではなく、**「貼り付け運用」で隣に置く**

### 解決する課題（営業トーク）

歯科医院では、カルテ記載に年間1,400時間（歯科医師4人体制シミュレーション）が消えている。
これが「見えない負担」として以下を引き起こしている：
1. 記録が追いつかず残業・持ち帰り
2. 記載の粒度が人でブレる（引継ぎ・監査・教育に支障）
3. 採用難で入力要員を増やせない
4. 記録が診療効率のボトルネック（1日に診られる患者数を制限）

## ✅ できること（主要8機能・実装済み）

### 01. 音声 → SOAP 自動変換
診療中の会話から S（主訴）・O（所見）・A（評価）・P（計画）を自動分類して記載。

**例：**
- S: 主訴 → 右上の奥歯がしみる
- O: 所見 → 冷水痛あり / う蝕疑い（E:6）
- A: 評価 → う蝕（C2）
- P: 計画 → う蝕除去 + CR充填・経過観察

### 02. 歯科専門用語辞書
歯式・病名・術式・材料名を正確に認識。**認識精度90%以上（自社検証）**。
Per（根尖性歯周炎）、Pul（歯髄炎）、CR（コンポジットレジン修復）、メタルインレー、メタルクラウン 等を区別可能。

### 03. 診療シーン別対応
口腔外科・矯正・予防・TC（トリートメントコーディネーター）など各シーンに対応。

### 04. 患者対応モニタリング
スタッフの患者対応を音声から確認し、接遇品質を平準化。

### 05. 患者検索
氏名・主訴・キーワードで「会話＋記録」を横断検索。

### 06. 音声データ安全保存
AES-256暗号化・法規制準拠の安全な保管。データは海外に一切送信しない。

### 07. カルテ整合性チェック
過去の記載内容との矛盾を自動検出。

### 08. カスタム辞書
院内ルール・独自用語を辞書に反映し、医院ごとの精度を向上。

## ❌ できないこと（誤解されやすい・絶対に成果物に書くな）

- **LINE連携・LINE公式アカウントからの自動応答・予約管理**
- **既存カルテシステム（レセコン）への直接書き込み連携**
  - SCHRAIBER は「貼り付け運用」が前提。書き戻しAPI連携はしない。
- **患者からの予約受付・問診チャット**
- **AIによる診断確定**（あくまでAIは「下書き」、最終判断は医師）
- **海外サーバーへのデータ送信**（AWS東京リージョン完結）

## 🛤️ 導入後のロードマップ（営業・CSで使う物語）

### Step 1: 記録（初期スタート） ─ まず安全に、入力負担を減らす
- 既存カルテに「貼り付け」で開始
- 基幹システム置換は不要
- 導入初日から使えるシンプルさ
- AI下書き → 医師確認 → 貼り付け

### Step 2: 標準化（教育・引継ぎ） ─ 院内ルールと知識を整備し、質を揃える
- カスタム辞書で院内ルール・用語を反映
- テンプレ適用で記録品質を均一化
- 新人教育・引継ぎの時間を大幅短縮
- PDCAで継続改善

### 目的
**運用が回るほど、効率化 → データ化 → 残業減 が加速する。**
