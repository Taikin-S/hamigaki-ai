## 🦷 SCHRAIBER.AI とは（コアコンセプト）

**一言で：** 歯科診療の「書く時間」を、ゼロにする AI音声カルテシステム。

- **形態：** 歯科診療に特化したクラウドAIサービス
- **入力：** 診療中の医師・スタッフの会話（音声）
- **出力：** SOAP形式の診療記録（テキスト）
- **使い方：** ①録音開始 → ②AI自動整形 → ③確認して既存カルテに貼り付け
- **思想：** 基幹カルテシステムを置き換えるのではなく、**「貼り付け運用」で隣に置く**

### 解決する課題（営業トーク）

歯科医院では、カルテ記載に年間1,400時間（歯科医師4人体制シミュレーション）が消えている。
これが「見えない負担」として以下を引き起こしている：
1. 記録が追いつかず残業・持ち帰り
2. 記載の粒度が人でブレる（引継ぎ・監査・教育に支障）
3. 採用難で入力要員を増やせない
4. 記録が診療効率のボトルネック（1日に診られる患者数を制限）

## ✅ できること（主要8機能・実装済み）

### 01. 音声 → SOAP 自動変換
診療中の会話から S（主訴）・O（所見）・A（評価）・P（計画）を自動分類して記載。

**例：**
- S: 主訴 → 右上の奥歯がしみる
- O: 所見 → 冷水痛あり / う蝕疑い（E:6）
- A: 評価 → う蝕（C2）
- P: 計画 → う蝕除去 + CR充填・経過観察

### 02. 歯科専門用語辞書
歯式・病名・術式・材料名を正確に認識。**認識精度90%以上（自社検証）**。
Per（根尖性歯周炎）、Pul（歯髄炎）、CR（コンポジットレジン修復）、メタルインレー、メタルクラウン 等を区別可能。

### 03. 診療シーン別対応
口腔外科・矯正・予防・TC（トリートメントコーディネーター）など各シーンに対応。

### 04. 患者対応モニタリング
スタッフの患者対応を音声から確認し、接遇品質を平準化。

### 05. 患者検索
氏名・主訴・キーワードで「会話＋記録」を横断検索。

### 06. 音声データ安全保存
AES-256暗号化・法規制準拠の安全な保管。データは海外に一切送信しない。

### 07. カルテ整合性チェック
過去の記載内容との矛盾を自動検出。

### 08. カスタム辞書
院内ルール・独自用語を辞書に反映し、医院ごとの精度を向上。

## ⚔️ 競合認識（営業・マーケが必ず把握しておくこと）

### 直接競合：既存カルテシステムが音声カルテ機能をサブで搭載してきた

- **SCO**（カルテシステム）
- **オプティック**（カルテシステム）
- **ジニー**（カルテシステム）
- 他にも既存カルテシステム各社が、サブカルテ機能として音声入力を追加してきている

### 構造的な戦況

**この市場でSCHRAIBER.AIが勝つ条件と負ける条件は明確：**

| 勝つ場面（SCHRAIBERが選ばれる） | 負ける場面（他社に流れる） |
|---|---|
| 代表（佐野泰喜）が歯科医師であること自体が決め手になる | 既にそのカルテシステム（SCO/オプティック/ジニー等）を導入済 |
| 歯科専門用語の認識精度に納得した | カルテ＋予約管理の一貫性を重視する医院 |
| | SCHRAIBERにまだ実績値・安心感が足りない |
| | プロダクトの不具合が現時点で残っている（要改善） |

### 戦略的な隙間（今の勝ち筋）

- **予約管理ツール（アポツール系）はまだ音声カルテ機能を出してきていない**
- ここに「予約管理は他社のまま、音声カルテだけSCHRAIBER」という棲み分けで入る余地がある
- 「まだ音声で認識できるカルテがない」予約管理ツールを使っている医院がコアターゲット

### 営業で握っておくべき敗北パターン（聞かれたら正直に話す）

- 既存カルテ会社のサブ機能で済まされる → 既存カルテに不満が薄い医院は無理に取りに行かない
- 予約管理との一貫性を求められる → SCHRAIBERは音声カルテに特化しているため、ここは正直に説明
- 実績・安心感への不安 → 既存導入医院（ママこども・リタ・二番町・湘南デンタルケア）の事例で補強

## 📣 営業の現状認識（正直版・全AIが踏まえること）

> **2026-05-12 時点の代表の自己認識：**
> 「まだ、私の説明勝ちな気がする。製品よりも期待感で売れている。」

### 何で売れているか
1. **佐野泰喜の人脈起点**：最初に佐野を知っている院長に話が届く構造
2. **臨床経験のある歯科医師が作った**という「理解している感」
3. 認識精度90%のデモ（補強材料）

### 何で売れていないか
- 製品の完成度（不具合が残っている）
- 実績値（導入医院数の少なさ）
- 製品単独の説得力（佐野の説明なしでは弱い）

### 戦略的な含意（マーケ・営業AIが必ず考慮）

| 含意 | アクション |
|---|---|
| 佐野の説明力に依存している | 説明を動画・コンテンツ・スライドに**仕組み化**して横展開する |
| 知り合いベースで限界が見える | 佐野を知らない院長にも届く「期待感の代替物」が必要（事例・登壇・PR） |
| 期待感で売れているが製品で繋ぎ止めねばならない | CSとプロダクトの強化が同時並行で必須 |
| 院長本人が動くプロダクト | コンテンツも提案書も**院長本人に直撃**する形式にする（事務長向けROIではなく、院長の精神的負担に訴える） |

### この情報を踏まえてAIがやってはいけないこと

- **「実績豊富」「業界トップ」など、実態と乖離した表現を使わない**
- **「製品の機能差」だけで競合に勝てる前提のコンテンツを作らない**（実際の勝ち筋は人と説明）
- **スタッフ・事務長向けのROI訴求を主軸にしない**（決裁者は院長本人）
