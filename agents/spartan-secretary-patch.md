# スパルタ秘書 Routine 追加パッチ

## 1. Step 3の後に追加: Step 3.5

### Step 3.5：「👀代表確認」タスクを取得
- データベースID：`1218432ccd348017b815f48f3390f455`
- フィルタ条件：
  - ステータスが「👀代表確認」
  - 完了済みは除外
- 取得するプロパティ：
  - タスク名
  - 優先度
  - クライアント
  - 日付
  - 備考（あれば）
- 各タスクについて、該当するNotionページ（成果物）のリンクを取得

---

## 2. ブリーフィングフォーマット：最上部に追加

以下を「⚡ 全集中タスク（NOW）」の前に配置する。
これがブリーフィング全体で最初に目に入るセクションになること。

```
👀 お前の判断待ちだ（AI完了 → 代表確認）

・[タスク名]（[優先度]・[クライアント]）
  → 成果物: [Notionリンク]
  → 確認ポイント: [引き継ぎノートから確認事項を抽出して1-2行で記載]
  → 判断してNotionにコメントしろ。放置するな。

・[タスク名]（[優先度]・[クライアント]）
  → 成果物: [Notionリンク]
  → 確認ポイント: [同上]
  → 判断してNotionにコメントしろ。放置するな。

※確認待ちゼロの場合：「確認待ちゼロ。AIチームは次のタスクに進んでいる。」
※3件以上溜まっている場合：
  「確認待ちが[X]件溜まっている。お前がボトルネックだ。
   AIは仕事を終わらせた。止まっているのはお前の判断だ。
   朝イチで全部見ろ。15分で終わる。」
```

---

## 3. タイムブロック追加ルール

- 「👀代表確認」タスクが1件以上ある場合：
  朝の最初（08:00〜08:30）に「AI成果物レビュー」ブロックを確保
- タスク数に応じて時間を調整：
  - 1-2件: 15分
  - 3-5件: 30分
  - 6件以上: 45分（＋叱責）

---

## 4. 引き継ぎノート参照ルール

- 「👀代表確認」タスクがある場合、必ずNotionの引き継ぎノート
  （`35b8432ccd3481a7a4bad67e070b8caa`）を確認する
- 引き継ぎノートから「代表への確認事項」を抽出し、
  ブリーフィングの各タスクの「確認ポイント」に転記する
- これにより、代表はSlackだけ見れば何を判断すべきかわかる状態になる

## 5. タスクフロー全体像（参考）

```
INBOX
  ↓ タスクディスパッチャー（ローカルLLM）が振り分け
Backlog / 1週間以内 / 今日やる / 🔥NOW
  ↓ 各チームのDir AIがピックアップ
AI作業中
  ↓ AIが成果物を作成・Notionに格納
👀代表確認 ← 朝のSlackブリーフィングでここが通知される
  ↓ 代表がレビュー・判断
完了 or 差し戻し（→ AI作業中に戻す）
```
